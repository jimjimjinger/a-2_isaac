## 3. Ground Truth X 마커 추가

**파일**: `isaac_drive/isaac_drive/raycast_map_viewer.py`

### 배경
- 기존 파란 삼각형은 `/rover/estimated_pose` (EKF 추정값) → 드리프트 있음
- 실제 절대좌표(Isaac Sim Ground Truth)를 맵 위에 별도로 표시해야  
  EKF 오차가 얼마나 되는지 시각적으로 확인 가능

### 구현

**추가된 구독 토픽**: `/ground_truth/odom` (`nav_msgs/Odometry`)

```python
# 임포트 추가
from nav_msgs.msg import OccupancyGrid, Odometry

# 파라미터
self.declare_parameter("gt_topic", "/ground_truth/odom")

# 구독
self.create_subscription(Odometry, gt_topic, self._on_gt_odom, 10)

# 상태 변수
self.gt_xy: Optional[tuple[float, float]] = None

# 콜백
def _on_gt_odom(self, msg: Odometry) -> None:
    p = msg.pose.pose.position
    with self.lock:
        self.gt_xy = (p.x, p.y)
```

**matplotlib 렌더링**:

## 5. 위치 보정 알고리즘 교체

**파일**: `isaac_drive/isaac_drive/raycast_map_viewer.py`

### 기존 방식: 단순 최근접 매칭 (`_compute_correction`)

```python
def _compute_correction(orig_list, ray_list, max_match_dist, max_correction_mag):
    diffs = []
    for rx, ry, _ in ray_list:
        # 각 raycast 장애물마다 가장 가까운 맵 장애물 하나 찾기
        best = min(orig_list, key=lambda m: hypot(m[0]-rx, m[1]-ry))
        if distance(best, ray) <= max_match_dist:
            diffs.append((best[0]-rx, best[1]-ry))
    return mean(diffs)  # 단순 평균
```

**문제점**:
- 노이즈 장애물이 엉뚱한 맵 장애물과 매칭되면 평균이 왜곡됨
- 각 쌍을 독립적으로 결정 → 전체 패턴의 일관성 없음
- 장애물 1-2개 상황에서 신뢰도 낮음

### 새 방식: 패턴 매칭 (`_pattern_match_correction`)

**핵심 아이디어**: 모든 (raycast_i, map_j) 앵커 쌍을 후보 shift로 시도,  
그 shift에서 나머지 장애물들까지 얼마나 일치하는지 채점(score)

```python
def _pattern_match_correction(orig_list, ray_list, match_thresh,
                               max_correction_mag, min_matches=2):
    best_score = 0
    best_dx, best_dy = 0.0, 0.0

    for (rx, ry, _) in ray_list:          # 앵커 raycast 장애물
        for (mx, my, _) in orig_list:     # 앵커 map 장애물
            dx_c = mx - rx
            dy_c = my - ry
            if hypot(dx_c, dy_c) > max_correction_mag:
                continue                  # 너무 큰 shift 제외

            # 이 shift에서 나머지 raycast 장애물들이 몇 개 일치하는지 채점
            match_dxs, match_dys = [], []
            used = set()
            for (rx2, ry2, _) in ray_list:
                sx, sy = rx2 + dx_c, ry2 + dy_c      # shift 적용
                best_k = 가장 가까운 미사용 map 장애물 인덱스
                if distance < match_thresh:
                    match_dxs.append(mx2 - rx2)
                    match_dys.append(my2 - ry2)

            score = len(match_dxs)
            if score > best_score:
                best_score = score
                best_dx = mean(match_dxs)   # 일치 쌍들 평균 → 더 정밀
                best_dy = mean(match_dys)

    if best_score < min_matches:
        return None, None, ...  # 실패

    return best_dx, best_dy, best_score, "ok"
```

### 패턴 매칭이 강건한 이유

```
예시: raycast 장애물 R1, R2, R3, R_noise 존재

앵커 (R_noise, M_엉뚱): shift = M_엉뚱 - R_noise
  → R1+shift: 맵에 없음 ✗
  → R2+shift: 맵에 없음 ✗
  → R3+shift: 맵에 없음 ✗
  score = 0 → 채택 안 됨

앵커 (R1, M1): shift = M1 - R1 = (+0.1, +0.1)
  → R2+shift: M2 근처 ✓
  → R3+shift: M3 근처 ✓
  → R_noise+shift: 맵에 없음 ✗
  score = 3 → 최고 score, 채택!
```
